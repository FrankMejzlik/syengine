12 /*===========================================================================
13  We assume that instructions are unsigned numbers.
14  All instructions have an opcode in the first 6 bits.
15  Instructions can have the following fields:
16  `A' : 8 bits
17  `B' : 9 bits
18  `C' : 9 bits
19  'Ax' : 26 bits ('A', 'B', and 'C' together)
20  `Bx' : 18 bits (`B' and `C' together)
21  `sBx' : signed Bx
22 
23  A signed argument is represented in excess K; that is, the number
24  value is the unsigned value minus K. K is exactly the maximum value
25  for that argument (so that -max is represented by 0, and +max is
26  represented by 2*max), which is half the maximum for the corresponding
27  unsigned argument.
28 ===========================================================================*/

219 /*===========================================================================
220  Notes:
221  (*) In OP_CALL, if (B == 0) then B = top. If (C == 0), then `top' is
222  set to last_result+1, so next open instruction (OP_CALL, OP_RETURN,
223  OP_SETLIST) may use `top'.
224 
225  (*) In OP_VARARG, if (B == 0) then use actual number of varargs and
226  set top (like in OP_CALL with C == 0).
227 
228  (*) In OP_RETURN, if (B == 0) then return up to `top'.
229 
230  (*) In OP_SETLIST, if (B == 0) then B = `top'; if (C == 0) then next
231  'instruction' is EXTRAARG(real C).
232 
233  (*) In OP_LOADKX, the next 'instruction' is always EXTRAARG.
234 
235  (*) For comparisons, A specifies what condition the test should accept
236  (true or false).
237 
238  (*) All `skips' (pc++) assume that next instruction is a jump.
239 
240 ===========================================================================*/
