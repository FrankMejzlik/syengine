50 // This snippet shows how to 'hide' the complexity of
51 // the STL by wrapping some useful piece of functionality
52 // around a handful of discrete API calls.
53 //
54 // This API allows you to create an indexed triangle list
55 // from a collection of raw input triangles. Internally
56 // it uses an STL set to build the lookup table very rapidly.
57 //
58 // Here is how you would use it to build an indexed triangle
59 // list from a raw list of triangles.
60 //
61 // (1) create a 'VertexLookup' interface by calling
62 //
63 // VertexLook vl = Vl_createVertexLookup();
64 //
65 // (2) For each vertice in each triangle call:
66 //
67 // unsigned int i1 = Vl_getIndex(vl,p1);
68 // unsigned int i2 = Vl_getIndex(vl,p2);
69 // unsigned int i3 = Vl_getIndex(vl,p3);
70 //
71 // save the 3 indices into your triangle list array.
72 //
73 // (3) Get the vertex array by calling:
74 //
75 // const float *vertices = Vl_getVertices(vl);
76 //
77 // (4) Get the number of vertices so you can copy them into
78 // your own buffer.
79 // unsigned int vcount = Vl_getVcount(vl);
80 //
81 // (5) Release the VertexLookup interface when you are done with it.
82 // Vl_releaseVertexLookup(vl);
83 //
84 // Teaches the following lessons:
85 //
86 // How to wrap the complexity of STL and C++ classes around a
87 // simple API interface.
88 //
89 // How to use an STL set and custom comparator operator for
90 // a complex data type.
91 //
92 // How to create a template class.
93 //
94 // How to achieve significant performance improvements by
95 // taking advantage of built in STL containers in just
96 // a few lines of code.
97 //
98 // You could easily modify this code to support other vertex
99 // formats with any number of interpolants.
100 //
101 // Hide C++ classes from the rest of your application by
102 // keeping them in the CPP and wrapping them in a namespace
103 // Uses an STL set to create an index table for a bunch of vertex positions
104 // used typically to re-index a collection of raw triangle data.
